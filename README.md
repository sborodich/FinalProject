# 🎭 Playwright Automation Framework

Финальный проект по автоматизации тестирования веб-приложения на фреймворке Playwright (JavaScript) с использованием современных паттернов проектирования.

Технологии использованные в проекте
* Движок тестов: Playwright
* Язык программирования: JavaScript
* Архитектурный паттерн: Page Object Model (POM)
* Непрерывная интеграция: GitHub Actions (CI/CD)

Покрытие тестами (10 сценариев)
1. Тесты 1-2 (DDT): Параметризованная проверка успешной регистрации и входа для пула пользователей.
2. Тест 3 (Негативный): Авторизация с некорректным паролем и валидация сообщения об ошибке.
3. Тест 4 (Негативный): Валидация блокировки отправки формы регистрации при пустом поле Email.
4. Тест 5 (Позитивный): Поиск товара в каталоге по ключевому слову.
5. Тест 6 (Позитивный): Добавление первого доступного товара в корзину.
6. Тест 7 (Позитивный): Удаление товара из корзины и проверка сообщения об очистке.
7. Тест 8 (Позитивный): Успешная подписка на рассылку новостей в футере сайта с динамическим Email.
8. Тест 9 (Позитивный): Прямой переход на страницу каталога товаров ("All Products").
9. Тест 10 (Негативный): Попытка отправки формы обратной связи без заполнения обязательного поля Email.

Настройки и стабильность
* Кроссбраузерность: Тесты настроены на параллельный запуск в Chromium, Firefox, WebKit и Mobile Chrome.
* Блокировка рекламы: Внедрен перехват сетевых запросов `route.abort()` и прямая навигация по URL для обхода всплывающих баннеров Google Ads, что исключает Flaky-падения по таймаутам.
* Чистый репозиторий: Настроен `.gitignore` для исключения тяжелых локальных отчетов и папок зависимостей.
* CI/CD: Настроен пайплайн в GitHub Actions с автоматическим прикреплением HTML-отчетов `Artifacts` к каждой сборке.
